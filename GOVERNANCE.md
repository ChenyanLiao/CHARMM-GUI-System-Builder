# Governance

## Canonical Project

The canonical upstream is:

<https://github.com/ChenyanLiao/CHARMM-GUI-System-Builder>

Forks and modified versions are permitted by AGPLv3, but they are not canonical
releases unless merged and released from this repository.

## Roles

- **Founder and Lead Maintainer:** Liao Chenyan (`@ChenyanLiao`)
- **Maintainers:** contributors explicitly appointed by the Founder and listed
  in this file through a reviewed pull request
- **Contributors:** anyone whose pull request is accepted

The Founder controls maintainer appointment, repository administration, release
signing, and the final decision to merge into protected branches. Maintainers
may review and merge only within permissions explicitly granted to them.

## Change Process

1. Open an issue for substantial behavioral or scientific changes.
2. Submit changes through a pull request from a branch or fork.
3. Pass the full test suite and security checks.
4. Obtain Code Owner approval for protected files.
5. Record new reusable failure modes without credentials, live job access, or
   workstation-specific paths.

## Releases

Only tags and releases published from the canonical repository by the Founder
or a delegated release maintainer are canonical releases. A fork may publish
its own releases, but must identify them as modified and non-canonical.
