# OpenClaw Adapter

## Installation Root

OpenClaw can load workspace, project-agent, personal, or managed skills. Use a
single installed copy named `charmm-gui-system-builder`; do not edit the
canonical Git checkout in place during a run.

## Tool Mapping

Fill the run-local runtime capability manifest before delegation so host-local
file, browser, download, and vault capabilities are explicit.

- Use the configured terminal/exec capability for bundled validators.
- Run the Guided Decision Protocol and verify the locked contract hash before
  delegating any browser or official API action.
- Use OpenClaw browser control only when it can preserve the user-authenticated
  session and expose the scientific form state required by the core workflow.
- Use a human operator for login, CAPTCHA, MFA, OS dialogs, and any upload/save
  dialog the browser tool cannot reach.
- If the OpenClaw agent runs remotely, confirm that local input and download
  paths actually exist on that host before acting.
- A remote agent may consume an already scoped, unexpired authorization but may
  not mint a replacement or receive the underlying login/signing secrets.

## Multi-Agent Handoff

When another agent operates the browser, give it only the contract hash, exact
job ID, step, route, maturity, resume URL, one allowed action, required
pre-submit fields, submission lock, and stop conditions. Never transfer
credentials or session artifacts between agents.
